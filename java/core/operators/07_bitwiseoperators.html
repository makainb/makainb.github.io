    # 位运算符

    # Java位运算符简介

    # 什么是位运算符

    对于二进制操作的叫位运算符，一个字节在(byte)Java中是8位(bit)，也就是8个二进制数。

    # Java中的七种位运算符

    <<   左移
>>   右移
>>>  无符号右移
&    与
|    或
~    非
^    异或

    其中非（~）是一元运算符，其他六个都是二元运算符。 这些位运算符都是作用在二进制的数上的，先列一个表描述一下这几种位运算符：

    # 或 （ | ）

    或运算 英语是 or ，符号是 | ** 。**两位都为0时，结果为0，样例：

    int a = 5; // 0101 (前面都是0，全部省略)
int b = 7; // 0111
System.out.println("a|b = " + (a | b)); 

运算过程如下:
  0101
| 0111
---------
  0111  = 7

    # 与（ & ）

    与运算 英语是 and ，符号是 & ** 。**两位都为1时，结果为1，样例：

    int a = 5; // 0101 (前面都是0，全部省略)
int b = 7; // 0111
System.out.println("a&b = " + (a & b));

运算过程如下:
  0101
& 0111
---------
  0101  = 5

    # 异或（ ^ ）

    异或 英语是 xor ，符号是 ^ ** 。**相同为0，不同为1，样例：

    int a = 5; // 0101 (前面都是0，全部省略)
int b = 7; // 0111
System.out.println("^b = " + (a ^ b));

运算过程如下:
  0101
^ 0111
---------
  0010  = 2

    # 非（ ~ ）

    非 英语是 Complement ，符号是 ~ 。取相反，0 变 1， 1 变 0，样例：

    int a = 5; // 0101 (前面都是0，全部省略)
System.out.println(~a);

运算过程如下:
~ 0101
---------
  1010  = 10 （你的结果跟我的结果不一致，我只显示了4位，实际上int是四个字节，32位，得出的符号位为1，结果为-6）

    # 右移（ >> ）

    右移 英语是 我不知道 ，符号是 >> ** 。**二进制全部往右移动N位，如果移动前最高位，原样补位，样例：

    样例 1:
int a = 10; // 1010
System.out.println(a>>1); //5

运算过程如下：
  1010
--------------
  0101

样例 2:
int a = -10;              // 11111111 11111111 11111111 11110110
System.out.println(a>>1); // 整体右移一位，结果是-5

  11111111 11111111 11111111 11110110
---------------------------------------------------
  11111111 11111111 11111111 11111011

    # 无符号右移（ >>> ）

    无符号右移 英语是 我不知道 ，符号是 >>> ** **。二进制全部往右移动N位，高位补0，样例：

    样例 1:
int a = 10;    // 1010
int a>>>1      // 0101

运算过程如下：
  1010
--------------
  0101

样例 2:
int a = -10 ; // 11111111 11111111 11111111 11110110
int a>>>1; // 2147483643
System.out.println(a>>>1); // 整体无符号右移一位，结果是2147483643

  11111111 11111111 11111111 11110110
---------------------------------------------------
  01111111 11111111 11111111 11111011

    # 左移（ << ）

    左移 英语是 Left Shift ，符号是 << ** **。二进制左移右干位，高位丢弃，低位补0，样例：

    样例 1:
int a = 5;    // 0000 0101
int a << 2    // 0001 0100

运算过程如下：
  0000 0101
--------------
  0001 0100

    顺便告诉大家个秘密，还可以写成省略形式，比如 a &= b 。

    # 运位算符的使用技巧

    # 判断奇偶数

    我们可以利用 & 运算符的特性，来判断二进制数第一位是0还是1。 用 if ((a & 1) == 0) 代替 if (a % 2 == 0) 来判断 a 是不是偶数。

    # 数字交换

    int a = 5;
int b = 10;
a ^= b;
b ^= a;
a ^= b;

    # 取余

    其实取余算法和上面的判断奇偶数原理是一样的。 比如说我们要让a对16进行取余，那么就可以让 a & 15 得出来的结果就是余数。 可以看出15的二进制表示为：

    0000 0000 0000 0000 0000 0000 0000 1111

    所以 a & 15 返回值就是a二进制的最低四位，也就是 a & 15 = a / 16。 使用 & 来进行取余的算法比使用 / 效率高很多，虽然只能对2^n的数值进行取余计算，但是在JDK源码中也是经常被使用到，比如说HashMap中判断key在Hash桶中的位置。

    # 生成第一个大于a的满足2^n的数

    这个标题可能显得不那么容易理解，下面结合场景来解释一下。 在HashMap中我们需要生成一个Hash桶，用来存储键值对（或者说存储链表）。 当我们查询一个key的时候，会计算出这个key的hashCode，然后根据这个hashCode来计算出这个key在hash桶中的落点，由于上面介绍的使用 & 来取余效率比 / 效率高，所以HashMap中根据hashCode计算落点使用的是 & 来取余。 使用 & 取余有一个局限性就是除数必须是2^n，所以hash桶的size必须是2^n。 由于HashMap的构造器支持传入一个初始的hash桶size，所以HashMap需要对用户传入的size进行处理，生成一个第一个大于size的并且满足2^n的数。 这个算法的使用场景介绍完毕了，那么再来看一下算法实现：

    # 循环判断

    public static final int tableSizeFor(int cap) {
    int size = 1;
    while (size < cap) {
        size *= 2;
    }
    return size;
}

    # | 运算符实现

    public static final int tableSizeFor(int cap) {
    int n = cap - 1;
    n |= n >>> 1;
    n |= n >>> 2;
    n |= n >>> 4;
    n |= n >>> 8;
    n |= n >>> 16;
    return (n < 0) ? 1 : (n >= MAXIMUM_CAPACITY) ? MAXIMUM_CAPACITY : n + 1;
}

    HashMap就是使用这个算法来修改用户使用构造器传进来的size的，这个算法是使用移位和或结合来实现的，性能上比循环判断要好。

    # 求相反数

     ~a + 1

    # 求绝对值

    a >> 31 == 0 ? a : (~a + 1)

    # 取一半的值

    System.out.println(5 >> 1); // 如果是奇数先减一，再除2
System.out.println(8 >> 1);// 如果是偶尔，直接除2
